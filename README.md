# ML Pipeline with CI/CD & Drift Monitoring — Customer Churn

> **Status: under construction.** Full architecture docs, quickstart, and a
> **Measured Results** table (populated exclusively from `eval/results/*.json`)
> land in Phase 6. No metric appears here unless it was read from a raw
> results file generated by this pipeline.

Customer churn prediction (Telco dataset) with:

- **Training & registry:** XGBoost + Optuna, tracked in MLflow on DagsHub, promotion-gated model registry
- **Serving:** FastAPI on Render free tier, with in-app shadow (champion/challenger) routing
- **Orchestration:** Airflow DAG (local Docker Compose) — drift-triggered retraining loop
- **Validation & drift:** Great Expectations + Evidently (PSI)
- **CI/CD:** GitHub Actions → GHCR → Render deploy hook, with an AUC quality gate
- **Monitoring:** Prometheus scraping the live `/metrics` endpoint + Grafana dashboard (local)

**Architecture split (deliberate):** orchestration + monitoring run on a separate
host (local Docker Compose); only the slim serving container deploys to Render.
This mirrors real production separation and respects Render free-tier limits
(512 MB RAM, spin-down on idle).
